## Carousel ARIA pattern (canonical)

Pattern: **APG carousel with manual + auto-advance**, slide group as `role="region"` with `aria-roledescription="carousel"`.

```html
<section aria-roledescription="carousel" aria-label="Featured engineers">
  <div aria-live="polite" aria-atomic="true" class="sr-only" id="hero-cv-live">
    Slide 2 of 3: Aisha Rahman, Chief Software Engineer
  </div>

  <button aria-label="Previous engineer" aria-controls="hero-cv">…</button>
  <button aria-label="Next engineer"    aria-controls="hero-cv">…</button>

  <ul id="hero-cv" role="presentation">
    <li role="group"
        aria-roledescription="slide"
        aria-label="2 of 3"
        aria-current="true">
      …
    </li>
  </ul>

  <div role="tablist" aria-label="Choose engineer">
    <button role="tab" aria-selected="true"  aria-controls="hero-cv">1</button>
    <button role="tab" aria-selected="false" aria-controls="hero-cv">2</button>
    <button role="tab" aria-selected="false" aria-controls="hero-cv">3</button>
  </div>
</section>
```

- Auto-advance **pauses** when any descendant has focus or hover.
- Auto-advance **disabled** under `prefers-reduced-motion`.
- Live-region copy template: `Slide {i} of {n}: {name}, {role}`. Updated **only** on slide change, never on auto-init.

## Focus order (DOM order matches visual order)

1. Skip link → `Skip to main content`
2. Eyebrow (non-interactive, skipped)
3. H1 (passthrough, non-interactive)
4. Primary CTA
5. Secondary CTA
6. Stat chip 1, 2, 3 (non-interactive, skipped)
7. Carousel "Previous"
8. Tab 1, Tab 2, Tab 3 (roving tabindex)
9. Carousel "Next"
10. Portrait LinkedIn icon button (if present)
11. Profile-card LinkedIn link

Verified via `axe` + `@axe-core/playwright` `focus-order-semantics`.

## Calendly modal — focus management

- On open: store `triggerRef = document.activeElement`. Move focus to modal close button. Trap focus inside modal.
- On close: `triggerRef.focus({ preventScroll: true })`. `aria-live` announces `Booking modal closed.` (cleared after 3s).
- ESC closes. Backdrop click closes only if not in mid-booking.
