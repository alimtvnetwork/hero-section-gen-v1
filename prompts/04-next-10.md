# 04 — The "Next 10" Loop

**Trigger:** User says "next 10" (or "next 10 with reasoning and time").

**Goal:** Execute the next 10 outstanding items from `45-priorities.md` in a
single batch, in priority order (P0 first), with a short reasoning line and
time estimate per item.

## Rules

1. Read `45-priorities.md` and pick the **next 10 items** with status
   `todo`, ordered P0 → P1 → P2, then by file appearance.
2. For each item, before editing, write a one-line reasoning + estimate:
   `#NN · <file> · <P0/P1/P2> · <one-line reason> · ~Xm`.
3. Create or edit the spec files. New files MUST follow the lowercase-hyphen
   numeric-prefix convention from `prompts/00-index.md`.
4. Update `00-index.md` if a new file is added (insert in the right slot,
   never append blindly).
5. Bump `21-changelog.md` with a single batch entry (one bullet per item).
6. Mark each completed item in `45-priorities.md` as `done` with the date.
7. End the reply with **"Remaining items: <N>"** and a 5-line preview of
   what the next batch will likely cover.

## Output template (in chat)

```
Batch <K> · items <X..X+9> · spec score <before> → <after>

1. <file> · P0 · <reason> · ~Xm — <one-line outcome>
...
10. ...

Remaining items: <N>
Next batch preview: <topic1>, <topic2>, <topic3>...
```

## Anti-patterns

- Doing more than 10. If item 10 is mid-feature, finish it and stop.
- Skipping reasoning to save tokens — the user explicitly wants it.
- Editing files outside the spec without a paired entry in `45-priorities.md`.
- Forgetting to update `00-index.md` after adding a file.

## Stop conditions

- `45-priorities.md` has 0 P0 + 0 P1 items left → reply "All P0/P1 done.
  Spec is at 100/100. Remaining P2 are polish."
- Any item requires the user → stop, ask, resume on answer.

## Deliverables per batch

- 10 spec items resolved.
- `21-changelog.md` bumped.
- `45-priorities.md` updated.
- `00-index.md` re-indexed if files were added.
